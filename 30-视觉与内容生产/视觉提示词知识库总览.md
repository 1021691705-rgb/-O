# 视觉提示词知识库总览

## 一句话总结

这套子知识库负责把视觉经验沉淀为可检索、可组合、可迭代的提示词规则、参考图反推模板和视觉词块，为生图、改图、电商详情页与商业视觉任务提供长期依据。

| 页面 | 作用 | 什么时候读 |
|---|---|---|

| [[30-视觉与内容生产/反推模板]] | 把参考图拆成结构化视觉描述和可继续写 Prompt 的视觉骨架 | 参考图分析、风格提取、详情页反推 |
| [[30-视觉与内容生产/视觉词块]] | 保存构图、镜头、光线、材质、版式、商业质感和改图控制词块 | 写 Prompt 或修正具体画面问题时检索 |

## 使用顺序

### 写生图 Prompt

```text
确认任务和产品约束
-> 阅读规则与规格中的相关章节
-> 分层检索视觉词块
-> 只采用与当前任务有关的词块
-> 按 AI生图 Prompt Skill 组织最终 Prompt
-> 生图后质检并沉淀可复用经验
```

### 反推参考图

```text
选择反推模板
-> 描述主体、构图、镜头、光线、材质和信息模块
-> 检查数量、位置、状态和视觉含义
-> 从视觉词块补充可复用机制
-> 输出视觉骨架，不直接复制品牌、价格和无关文字
```

### 改图和局部编辑

先读取 [[30-视觉与内容生产/规则与规格#9. Image Editing Rules]]，再检索 [[30-视觉与内容生产/视觉词块#C. Image Editing and Image-to-Image Control Phrases]]。明确写清“改什么”和“保持什么不变”。

## 检索方法

每次视觉任务按四层检索，不只搜索产品名或平台名：

1. 任务领域与主体类型，例如产品、人物、空间、电商海报。
2. 视觉机制，例如构图、镜头、光线、材质、排版、交互关系。
3. 质量控制，例如接触阴影、材质高光、文字可读性、边缘干净度。
4. 当前限制，例如产品不可变形、指定画幅、保留区域、禁止元素。

当前没有向量检索时，使用 Obsidian 搜索或 `rg`。检索命中只是入口，采用词块前仍要阅读上下文。

## 入库边界

- 默认沉淀可跨产品和项目复用的视觉机制，不保存完整项目 Prompt。
- 不保存客户隐私、真实密钥、价格、品牌专属内容和一次性偶然细节。
- 优先保存能控制画面行为的具体词块，不保存只有“高级、真实、好看”等宽泛形容词的空词块。
- 新词块先检查重复和近似表达，再合并到最接近的现有章节。
- 规则更新优先修改现有原则，不连续追加零散补丁。
