# 反推模板

> Reusable English templates for analyzing a reference image and turning it into prompt-ready visual descriptions. Do not store full finished project prompts here. Every reverse analysis should extract both visual structure and commercial image-quality mechanisms. Quality mechanisms must be written as concrete visual behavior, not generic quality adjectives: describe how the subject is grounded, how contact shadows touch the surface, how materials respond to light, how effects blend into the scene atmosphere, how text zones stay readable, how module edges stay clean, and what artifact risks must be avoided.

使用时先按任务类型选择模板，再从 [[30-视觉与内容生产/视觉词块]] 补充相关视觉机制，最终检查 [[30-视觉与内容生产/规则与规格]] 中的质量和边界要求。

## 1. General Reverse Prompt

```text
Analyze this image as an AI image-generation reference with maximum visual precision. Describe it in detail, including the main subject, subject position, scale, composition, camera angle, lighting, color palette, background, atmosphere, materials, visual style, and all important visible elements. For any layout, infographic, UI-like element, repeated object, icon group, label group, chart, indicator, comparison mark, or visual system, describe the exact number, position, arrangement, state, and meaning of each visible element. Also extract concrete commercial image-quality mechanisms as visual behavior: how the subject is grounded, how contact shadows touch surfaces, how reflections fade, how materials react to light, how atmosphere or effects blend into the scene, how text zones remain clean, how module edges stay crisp, and which artifacts would break realism. Focus on reusable visual language rather than brand names or copyrighted elements. End with one concise prompt-ready paragraph that preserves the same visual structure and quality direction.
```

## 2. Product Image Reverse Prompt

```text
Analyze this product image for AI image generation with maximum product and layout precision. Describe the product presentation, product position, scale, camera angle, composition, background, props, lighting, shadows, reflections, material texture, commercial photography style, and overall premium feeling. If the image includes icons, labels, annotation lines, feature callouts, status indicators, comparison modules, or repeated visual elements, describe the exact count, arrangement, position, state, and meaning of each one. Also describe concrete product-quality mechanisms: how the product base sits on the surface, how the contact shadow darkens at the touch point, how reflections fade across the tabletop, how transparent/glossy/matte materials catch light, how edges avoid cutout halos, whether the product feels photographic or CGI-like, and what quality controls should carry into later prompts. Do not copy brand names, logos, prices, or unrelated text. End with one prompt-ready paragraph that can guide a new product image with the same visual structure and commercial finish.
```

## 3. Poster and Layout Reverse Prompt

```text
Analyze this poster or commercial layout with maximum structural precision. Describe the visual hierarchy, headline placement, product or subject placement, text blocks, icons, labels, arrows, information modules, grid structure, spacing, background treatment, color palette, and overall advertising style. For every visible module, describe its exact position, size relationship, number of items, arrangement, state, and visual meaning. Also extract concrete layout-quality mechanisms: how text is kept on calm readable background zones, how cards use consistent corner radius and soft shadows, how icon strokes stay uniform, how callout lines connect cleanly, how product shadows and reflections match the layout, how spacing prevents crowding, and what artifacts would make modules look muddy or cheap. Separate visual structure from specific written content. End with one prompt-ready paragraph focused on layout, commercial visual direction, and quality controls.
```

## 4. Photography and Style Reverse Prompt

```text
Analyze this image as a photography and visual-style reference with maximum visible-detail precision. Describe the camera distance, lens feeling, depth of field, perspective, lighting setup, shadow quality, color grading, mood, styling, texture realism, retouching level, and overall photographic language. Also describe all visible props, repeated elements, subject-object relationships, foreground/midground/background details, and any visual structures that affect the final image. Extract the quality mechanisms that make the image believable or premium as concrete visual behavior: how focus transitions from sharp subject to softer background, how exposure protects highlights, how skin/product/material texture remains natural, how shadows attach to contact points, how atmospheric haze, mist, glow, dust, or motion effects blend with the light, how edges stay clean, and how post-processing remains restrained. End with one concise prompt-ready paragraph using professional photography vocabulary.
```

## 5. Detail Page Reference Reverse Prompt

```text
Analyze this e-commerce detail-page reference image with maximum structural detail. Describe the section purpose, subject placement, product scale, layout structure, headline position, text area distribution, icon or callout usage, information modules, bottom panels, charts, status indicators, legends, segmented bars, dots, numeric ranges, product-scene relationship, background, lighting, color system, spacing, and commercial design style. For every infographic or UI-like module, describe how many rows, columns, icons, dots, segments, labels, and what each visual state means. Focus on the reusable page framework and visual method, not the exact product, brand, price, or copy. End with one prompt-ready visual skeleton that preserves the same visual structure; this skeleton is a bridge for later prompt writing, not the final generation prompt.
```

## 6. Long Detail Page Reverse Prompt

Use this when the reference is a tall, multi-section e-commerce detail-page image rather than a single poster or single product scene.

```text
Analyze this tall e-commerce detail-page reference as a continuous conversion flow, not as one isolated poster. First identify why it is a long detail page: vertical multi-section structure, repeated product anchors, progressive selling points, dense feature modules, lifestyle proof scenes, operation/detail close-ups, bottom feature grids, or cross-section visual consistency. Then reverse-analyze it section by section.

For each visible section, describe its role in the page flow, such as hero hook, main product promise, lifestyle proof, functional proof, comparison/proof module, operation explanation, material/detail close-up, accessory/compatibility module, or bottom feature grid. For each section, describe the product position, product scale, camera angle, background scene, headline/text area, information modules, icon/card/count structure, callout lines, metric badges, props, lighting, color palette, and the relationship between product, scene, and selling point.

Also describe the full-page system: repeated product anchors, reading direction from top to bottom, section transitions, shared typography, shared icon/card language, shared color accents, consistent lighting/material treatment, and how the page moves from emotional hook to functional trust to purchase reassurance. Separate reusable mechanisms from one-off details. Do not copy brand names, prices, exact numbers, exact image text, or product-specific incidental details unless the user explicitly asks to preserve them.

Output with these headings: page type evidence, section map, product anchor system, typography and copy hierarchy, information-module system, scene and proof logic, lighting/material/color system, commercial image-quality mechanisms, reusable mechanisms, one-off details to avoid copying, and prompt-ready visual skeleton. The prompt-ready visual skeleton must be a concise visual blueprint for the later final prompt writer; it is not the final generation prompt and must not contain task metadata, file paths, model flags, or instructions like follow the reference.
```

## 7. Image-to-Image Style Extraction

```text
Analyze the reference image and extract only the reusable visual DNA: composition, camera angle, lighting, color palette, background style, material feeling, atmosphere, layout rhythm, commercial quality, and all visible structural details that should be preserved. For repeated objects, icons, labels, panels, indicators, or comparison elements, describe the exact count, arrangement, state, and meaning. Do not describe irrelevant objects that should not appear in the new image. End with a concise style-transfer paragraph.
```

## 8. Required Analysis Checklist

When reverse-prompting a reference image, check these points:

- `main subject`
- `subject position and scale`
- `composition type`
- `layout structure`
- `foreground, midground, background relationship`
- `camera angle and lens feeling`
- `lighting direction and shadow quality`
- `color palette and mood`
- `background and scene details`
- `materials and textures`
- `props and supporting elements`
- `typography and information hierarchy if present`
- `number of icons, labels, rows, columns, panels, and indicator states`
- `meaning of each infographic state, such as dots, bars, levels, percentages, or comparison marks`
- `bottom information panel structure if present`
- `commercial style and finish`
- `visible elements that must not be missed`
- `exact count of repeated elements, icons, labels, dots, bars, panels, props, and modules`
- `position and arrangement of every important visual element`
- `state and meaning of each visual indicator or comparison element`
- `icon visual style, stroke weight, shape language, internal symbol, and label relationship`
- `typography style, font weight, case, texture, edge treatment, distress level, spacing, and readability`
- `commercial image-quality mechanisms as concrete visual behavior: subject grounding, surface contact shadows, reflection fade, material-specific highlights, atmospheric/effect integration, clean edge treatment, readable text zones, crisp module boundaries, and artifact risks`
- `quality constraints worth carrying forward: avoid muddy low-resolution texture, fake CGI surfaces, inconsistent shadows, flat lighting, detached effects, unreadable modules, or over-processed HDR look`
- `for long detail pages: section count, section role, section transition, repeated product anchors, and top-to-bottom selling logic`
- `for long detail pages: which modules create emotional hook, functional proof, trust proof, operation explanation, and final reassurance`

## 9. Output Rule

- End with one prompt-ready visual skeleton or prompt-ready paragraph, depending on the selected template.
- Treat the prompt-ready skeleton as an intermediate visual blueprint for later prompt writing, not as the final generation prompt.
- Keep the output visual and reusable.
- Do not include `--ar` or model-specific command flags unless explicitly needed.
- Do not copy brand names, logos, prices, copyrighted characters, or exact image text unless the user explicitly asks to preserve text.
- Do not deposit specific image text into the knowledge base; deposit only reusable structure, such as headline position, icon column, status panel, or indicator scale.
- If the final prompt is in English, translate every important visual structure from the analysis into English.
- Core check: `If someone only reads the final prompt, can they imagine the same image structure as the reference?`
- Detail check: `Have all counts, positions, states, and meanings of visible modules been described?`
- Quality check: `Have the commercial finish been described as concrete visual mechanisms, such as product grounding, contact shadows, reflection fade, material highlights, atmospheric/effect integration, edge cleanliness, readable text zones, module clarity, and artifact risks?`
- Icon check: `Have the icon style, line weight, outer shape, internal symbol, and label relationship been described?`
- Typography check: `Have the font style, weight, case, texture, edge treatment, spacing, and readability been described?`
