# 知识入库 Skill

## 一句话总结

把用户发来的碎片知识、截图、链接、案例和经验，整理成可长期复用的 Obsidian Markdown 笔记，并在完成后按默认流程同步到 GitHub 知识库。

## 参考来源

- Obsidian 官方帮助：内部链接、标签、属性、图谱视图。
- Anthropic `skill-creator`：Skill 应该沉淀明确流程、触发条件和输出规范。
- Obsidian Git：本地知识库可通过 Git 同步到远端仓库。
- 本公司知识库路径：`C:\Users\HTX-GaoJie\Documents\obsidian 2`

## 触发场景

- 用户说“放进知识库”“记一下”“整理到 Obsidian”。
- 用户发截图、链接、短视频内容、项目经验、流程建议。
- 用户补充了之前待整理资料的标题、正文、截图或口播。

## 输入

- 原始内容：文本、链接、截图、图片、音视频转写。
- 来源信息：平台、作者、日期、链接。
- 用户意图：流程、岗位协同、降本增效、案例复盘、工具方法。
- 关联性判断：这条内容和知识库里哪 1-3 条已有笔记有关，关系是什么。
- 是否需要立即同步 GitHub：默认需要，除非用户明确说只本地保存。

## 输出

- 一篇 Obsidian Markdown 笔记。
- 笔记内包含：一句话总结、来源、核心观点、可执行方法、风险、下一步行动、相关链接。
- 更新对应总览或收集箱。
- 完成后进入 GitHub 同步流程。

## 标准流程

```text
接收资料
-> 判断是否可读
-> 提取核心信息
-> 补充待确认点
-> 判断与已有笔记的关联
-> 选择目录
-> 写入 Obsidian Markdown
-> 更新总览 / 反链
-> 新建 Git 分支
-> 提交并说明新增知识
-> 推送分支
-> 合并 main
-> 推送 main
```

## 目录建议

- `00-收集箱`：内容不完整、待补充。
- `30-资源/工具`：工具、教程、方法论。
- `30-资源/工具/Skill研究`：具体 Skill 设计与研究。
- `20-领域`：长期经营的领域知识。
- `10-项目`：具体项目过程和交付记录。

## 质量标准

- 不编造来源没有提供的信息。
- 无法读取链接时，先标记不可读，不强行总结。
- 每篇笔记至少有一个明确用途。
- 每篇笔记要能被后续项目复用。
- 链接到 1-3 条已有相关笔记，减少孤岛节点。
- 暂时只是想法的主题先写普通文字，不创建空链接。
