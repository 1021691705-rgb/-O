# 项目复盘 Skill

## 一句话总结

把已完成项目的过程、结果、返工原因和可复用经验沉淀成 SOP、检查清单或新的 Skill。

## 参考来源

- Atlassian 复盘 / retrospective / postmortem 方法：复盘要记录事实、影响、根因、改进项和责任人。
- AI设计公司总控目标：通过复盘减少返工，降低成本，提高协同效率。

## 触发场景

- 一个设计项目完成。
- 发生明显返工或质量问题。
- 用户想沉淀流程经验。
- 某个岗位 SOP 需要升级。

## 输入

- 项目目标。
- 最终交付物。
- 中间版本。
- 返工记录。
- 客户或内部反馈。
- 时间和成本消耗。

## 输出

- 项目复盘笔记。
- 有效做法。
- 问题和根因。
- 下次避免方法。
- 可复用 SOP。
- 可转化的新 Skill。

## 标准流程

```text
回顾目标
-> 对比结果
-> 列出做对的事
-> 列出返工和问题
-> 分析根因
-> 提炼 SOP
-> 更新相关 Skill
-> 同步知识库
```
