# 知识库使用说明书

## 一句话总结

这个知识库不是资料仓库，而是 AI设计公司的工作系统：用来收集知识、沉淀流程、调用 Skill、复盘项目，并把经验变成可重复使用的资产。

## 你每天怎么用

### 0. 在所有项目里怎么用

这个知识库可以作为所有 Codex 项目的长期资料库使用。

本机固定路径：

```text
C:\Users\HTX-GaoJie\Documents\obsidian 2
```

以后在其他项目里，如果你说“按知识库来”“查知识库”“入库”“开项目”“复盘”，Agent 默认应该先参考本知识库，再结合当前项目文件执行。

适合跨项目参考的内容：

- AI设计公司流程、岗位协同和项目复盘。
- Agent / Skill 设计、调用规则和防误用规则。
- 图片处理、电商详情页、AI 生图和质检流程。
- 飞书、企业微信、GitHub 同步等工具经验。

### 1. 有新资料时

先判断资料完整不完整。

资料不完整时，先放进 [[00-收集箱/收集箱]]：

```text
标题：
来源：
原始链接 / 截图：
一句话说明：
我想以后怎么用它：
还缺什么信息：
```

资料比较完整时，直接整理进对应目录：

- 工具、教程、方法论：放进 `30-资源/工具`
- Agent / Skill 经验：放进 `30-资源/工具/Skill研究`
- 长期领域知识：放进 `20-领域`
- 具体项目过程：放进 `10-项目`
- 每日零散记录：放进 `40-每日记录`

新增正式笔记时，默认问：

```text
这条笔记和知识库里哪 1-3 条已有笔记有关？
关系是什么？
```

找不到关联时，先放收集箱，不急着建正式笔记。

### 2. 想找资料时

优先从这几个入口找：

- [[首页]]：总入口，适合找当前重点。
- [[30-资源/资源总览]]：找工具、方法论、教程、概念资料。
- [[30-资源/工具/Skill研究/Skill研究总览]]：找可复用 Skill。
- [[30-资源/工具/Skill研究/Agent 读取知识库规则]]：给 Agent 判断该读哪些笔记。
- [[30-资源/工具/Skill研究/AI设计公司工作流地图]]：按工作流程找下一步该用什么。
- [[05-常用/常用入口]]：找高频工具和常用 Skill。

如果不知道关键词，从工作流开始找：

```text
资料进入 -> 知识入库
模糊需求 -> 设计项目开项
项目拆解 -> 岗位任务分配
详情页 -> 电商详情页方案
生图 -> AI生图 Prompt
图片处理 -> Photoshop 裁切导出
交付前 -> 图片质检
项目结束 -> 项目复盘
对外发布 -> 小红书图文生产
```

### 3. 想让 Codex 帮你入库时

可以直接这样说：

```text
把这段内容放进知识库，按知识入库流程整理。
```

如果你发的是链接或截图，可以补一句：

```text
重点从流程、岗位协同、降本增效、案例复盘、工具方法里选一个角度沉淀。
```

如果你只想临时保存：

```text
先放收集箱，不要整理成正式笔记。
```

如果你想整理成正式资产：

```text
整理成正式笔记，并补充关联笔记和下一步行动。
```

### 4. 想使用已有 Skill 时

先去 [[30-资源/工具/Skill研究/AI设计公司工作流地图]] 判断你处在哪个阶段，再打开对应 Skill。

常见用法：

- 需求很模糊：用 [[30-资源/工具/Skill研究/设计项目开项 Skill]]
- 要拆岗位任务：用 [[30-资源/工具/Skill研究/岗位任务分配 Skill]]
- 要做详情页结构：用 [[30-资源/工具/Skill研究/电商详情页方案 Skill]]
- 要写生图提示词：用 [[30-资源/工具/Skill研究/AI生图 Prompt Skill]]
- 要裁切导出图片：用 [[05-常用/Photoshop 裁切导出 Skill]]
- 要检查图片质量：用 [[30-资源/工具/Skill研究/图片质检 Skill]]
- 项目做完了：用 [[30-资源/工具/Skill研究/项目复盘 Skill]]
- 要发小红书：用 [[30-资源/工具/Skill研究/小红书图文生产 Skill]]

### 5. 怎么避免用错相似 Skill

Skill 多起来以后，不要只说“用那个图片 Skill”或“用设计 Skill”。最好说清楚四件事：

```text
用【准确 Skill 名称】帮我做【具体任务】
输入：【图片 / 文件 / 链接 / 文本 / 路径】
要求：【尺寸、格式、风格、限制、不能做什么】
输出：【你想拿到什么结果】
```

例如：

```text
用【Photoshop 裁切导出 Skill】帮我裁商品图。
输入：这张图片
要求：裁成 1000x1000，产品完整，居中，白底，不要拉伸
输出：JPG 文件
```

当存在多个相似 Skill 时，默认按下面顺序判断：

1. 你明确点名的 Skill 优先。
2. 如果点名和任务明显冲突，先提醒你确认。
3. 如果你没点名，就按 [[30-资源/工具/Skill研究/Agent 读取知识库规则]] 和 [[30-资源/工具/Skill研究/AI设计公司工作流地图]] 判断当前阶段。
4. 如果两个 Skill 都像能用，先比较它们的“触发场景”和“输出”。
5. 如果仍不确定，先问你，不直接执行。

常见容易混淆的情况：

| 你要做的事 | 应该优先用 |
|---|---|
| 规划详情页每一屏怎么卖 | [[30-资源/工具/Skill研究/电商详情页方案 Skill]] |
| 把已确认画面转成生图提示词 | [[30-资源/工具/Skill研究/AI生图 Prompt Skill]] |
| 裁图、方图、导出 JPG | [[05-常用/Photoshop 裁切导出 Skill]] |
| 检查图片有没有问题 | [[30-资源/工具/Skill研究/图片质检 Skill]] |
| 需求还很模糊，需要开项目 | [[30-资源/工具/Skill研究/设计项目开项 Skill]] |

如果你想更稳，可以直接加一句：

```text
如果有相似 Skill，先告诉我你准备选哪个和为什么。
```

### 6. 项目怎么记录

项目有明确结束条件，放进 [[10-项目/项目总览]]。

项目笔记至少记录：

```text
项目目标：
交付物：
当前资料：
缺失资料：
涉及岗位：
下一步：
风险：
复盘入口：
```

项目完成后，不要只写“完成了”，要补：

- 做对了什么
- 返工原因是什么
- 哪些流程可以复用
- 哪些内容应该变成 SOP 或 Skill

### 7. 什么时候同步 GitHub

适合同步：

- 新增正式笔记
- 更新总览入口
- 修改 Skill 流程
- 完成一轮知识库整理
- 修复断链、空节点、重复入口

不适合同步：

- 只有 Obsidian 图谱视图变化
- 未命名 canvas 草稿
- workspace 状态
- 临时截图、未确认资料
- 密钥、token、cookie、私密配置

同步前先用 [[30-资源/工具/Skill研究/知识库健康检查清单]] 检查：

- 没有 0 字节 Markdown
- 没有重复同名笔记
- 没有断链
- 没有空链接占位
- 没有旧路径
- Git 状态只包含本次确认要同步的内容

## 推荐的日常节奏

### 每天

- 新资料先进收集箱。
- 临时想法写进每日记录。
- 真正可复用的经验再整理成正式笔记。

### 每周

- 清理一次收集箱。
- 看一次 [[30-资源/工具/Skill研究/AI设计公司工作流地图]]，确认流程有没有断点。
- 把完成项目补进复盘。
- 跑一次 [[30-资源/工具/Skill研究/知识库健康检查清单]]。

### 每次项目结束

- 补项目复盘。
- 把返工原因变成检查项。
- 把稳定流程变成 Skill。
- 把高频入口放进 [[05-常用/常用入口]]。

## 最重要的规则

- 先收集，再整理，不要一开始就追求完美。
- 一个主题只保留一个主入口。
- 总览页只做导航，不堆长内容。
- 正式笔记必须有用途、来源、关联和下一步。
- 能复用的流程，最后都应该沉淀成 Skill。
