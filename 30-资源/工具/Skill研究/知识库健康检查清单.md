# 知识库健康检查清单

## 一句话总结

用一组固定检查确认 Obsidian 知识库没有空文件、重复入口、断链、无关本地状态和同步边界问题。

## 触发场景

- 完成一次知识入库或知识库整理后。
- 准备同步 GitHub 前。
- 图谱里出现空节点、孤岛节点或重复主题时。
- 用户说“整理一下”“同步前检查”“搞到最佳状态”。

## 检查项

| 检查项 | 合格标准 |
|---|---|
| 空 Markdown | 没有 0 字节 `.md` 文件 |
| 重复同名笔记 | 没有同一标题散落在多个目录 |
| Obsidian 断链 | 双中括号链接都能指向现有笔记 |
| 空链接占位 | 没有空的双中括号链接或“空链接”占位 |
| 关联性 | 正式笔记能连接到 1-3 条相关笔记 |
| 总览入口 | 首页、资源总览、Skill 总览能进入核心流程 |
| 收集箱 | 待整理内容有状态，已整理内容有去向 |
| 同步边界 | 不提交图谱状态、workspace、canvas 草稿、密钥 |

## PowerShell 检查命令

```powershell
$root = "C:\Users\HTX-GaoJie\Documents\obsidian 2"

# 空 Markdown
Get-ChildItem -Path $root -Recurse -Filter *.md |
  Where-Object { $_.Length -eq 0 } |
  ForEach-Object { $_.FullName.Substring($root.Length + 1) }

# 重复同名笔记
Get-ChildItem -Path $root -Recurse -Filter *.md |
  Group-Object BaseName |
  Where-Object { $_.Count -gt 1 }

# 空链接占位和旧路径
rg -n "D:\\ObsidianVault|\[\[\]\]" $root
```

## Git 同步前检查

```bash
git status --short --untracked-files=all
git diff --name-status
```

同步时只添加本轮确认过的内容文件。看到这些内容时要避开：

- `.obsidian/graph.json`
- `.obsidian/workspace.json`
- `.obsidian/workspace-mobile.json`
- `*.canvas`
- 密钥、token、cookie
- 未确认来源的资料草稿

## 已处理的边界

- `.obsidian/graph.json` 已加入 `.gitignore`，并从 Git 索引移除，只保留本地使用。
- `*.canvas` 已加入 `.gitignore`，未命名 canvas 草稿不会进入同步。
- `.gitattributes` 已固定 Markdown / JSON 等文本文件换行规则。
