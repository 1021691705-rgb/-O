# AI生图 Prompt Skill

## 一句话总结

把已确认的设计方案、产品约束和画面 brief 转成稳定、可控、可复用的 AI 生图 prompt。

## 参考来源

- OpenAI 图像生成文档：提示词应清楚描述主体、风格、构图、光线、材质和输出需求。
- AI设计公司规则：AI生图师不能自行改方案，只能把已确认方案转成 prompt。

## 触发场景

- 用户要求 `/生图`。
- 已有创意方向，需要生成主图、详情页场景图、封面或小红书图文。
- 上一轮图片跑偏，需要重写 prompt。

## 输入

- 已确认方案。
- 产品结构约束。
- 画面 brief。
- 尺寸和平台。
- 参考图。
- 禁止项。

## 输出

- 正向 prompt。
- 负面约束。
- 尺寸参数。
- 重试策略。
- 质检关注点。

## 知识库调用

正式写 Prompt 前，先读取 [[30-资源/视觉提示词知识库/规则与规格]] 中与任务相关的章节，再按以下四层检索 [[30-资源/视觉提示词知识库/视觉词块]]：

1. 任务领域与主体类型。
2. 构图、镜头、光线、材质、版式等视觉机制。
3. 商业质感和常见瑕疵的质量控制。
4. 当前产品、画幅、文字、参考图和禁止项约束。

有参考图时，先从 [[30-资源/视觉提示词知识库/反推模板]] 选择对应模板，得到视觉骨架后再写最终 Prompt。只采用与当前任务有关的词块，不机械堆叠检索结果。

## Prompt 结构

```text
主体：
产品结构：
场景：
构图：
镜头：
光线：
材质：
色彩：
风格：
平台尺寸：
禁止项：
```

## 质量标准

- 产品结构不能变形。
- 材质、尺寸、部件不能凭空改。
- 画面要服务卖点，不只追求好看。
- 负面约束要明确。
- 每个 prompt 都能被下一轮修改复用。
