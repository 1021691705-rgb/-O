# AI设计公司工作流地图

## 一句话总结

把 AI设计公司的知识入库、设计执行、交付质检、项目复盘和内容分发串成一条可重复执行的流程。

## 主流程

```text
外部资料 / 经验 / 截图 / 链接
-> Agent 读取知识库
-> 知识入库
-> Obsidian 笔记整理
-> GitHub 知识库同步

模糊设计需求
-> 设计项目开项
-> 岗位任务分配
-> 电商详情页方案
-> AI生图 Prompt
-> Photoshop 裁切导出
-> 图片质检
-> 项目复盘

可公开经验
-> 小红书图文生产
```

## 关联性检查

新增或整理笔记时，默认问：

```text
这条笔记和知识库里哪 1-3 条已有笔记有关？
关系是什么？
```

如果找不到关联，先放在收集箱；如果关联太多，优先补到总览页或工作流地图。

## 使用规则

- 工作流地图只放入口和关系，不堆具体操作细节。
- 具体操作写进对应 Skill 笔记。
- 尚未成文的未来主题先写成普通待办，不急着创建空链接。
- 高频入口可以放进 [[05-常用/常用入口]]。
