# 岗位任务分配 Skill

## 一句话总结

把设计项目拆给正确岗位，明确每个岗位的任务、输入、输出、完成标准和交接对象。

## 参考来源

- RACI / 项目角色分工方法：明确负责人、协作者、验收者。
- AI设计公司总控岗位体系：项目经理、创意总监、平面设计师、3D设计师、建模设计师、AI生图师、质检总监、流程优化经理等。

## 触发场景

- 用户使用 `/分配任务`。
- 项目已经开项，需要开始执行。
- 多岗位协同不清楚，容易重复劳动或漏项。

## 输入

- 项目档案。
- 交付物清单。
- 已有资料。
- 缺失资料。
- 时间要求。
- 岗位可用情况。

## 输出

- 任务拆解原则。
- 任务分配表。
- 关键拍板点。
- 每个岗位的验收标准。
- 下一步执行顺序。

## 标准流程

```text
确认项目目标
-> 拆交付物
-> 匹配岗位
-> 定义输入和输出
-> 定义完成标准
-> 明确交接对象
-> 标注风险和依赖
```

## 任务分配表字段

| 岗位 | 任务 | 需要资料 | 输出物 | 完成标准 | 交接对象 |
|---|---|---|---|---|---|
| 项目经理 | 统筹项目 | 需求、时间、资料 | 项目看板 | 任务清晰可执行 | 全岗位 |
| 创意总监 | 拍板方向 | 品牌、参考、用户 | 视觉策略 | 风格统一可执行 | 设计岗位 |
| AI生图师 | 转 prompt | brief、约束、尺寸 | 生图 prompt | 可复用可控 | 质检总监 |
| 质检总监 | 检查质量 | 原需求、方案 | 质检意见 | 能指导返工 | 项目经理 |
