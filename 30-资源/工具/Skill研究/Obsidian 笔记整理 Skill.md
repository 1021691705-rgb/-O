# Obsidian 笔记整理 Skill

## 一句话总结

把 Obsidian 中零散、重复、空链接和待整理内容，归并成结构清楚、可搜索、可关联、可同步的知识库。

## 参考来源

- Obsidian 官方帮助：内部链接、反向链接、标签、属性和图谱视图。
- Obsidian 官方说明强调链接和图谱用于发现笔记之间的关系。
- 本次经验：空链接会在图谱里形成节点，但点进去没有内容，影响知识库可用性。

## 触发场景

- 图谱里出现大量空节点。
- 同一主题拆成多篇重复笔记。
- 收集箱中有已整理但未迁移的条目。
- 用户说“这两个应该是同一个知识库的，合并一下”。

## 输入

- 目标笔记路径。
- 相关空链接或重复笔记名。
- 当前总览文件。
- 用户希望保留的主笔记。

## 输出

- 合并后的主笔记。
- 删除或归档重复笔记。
- 更新总览和反链。
- 清除失效 `[[空链接]]`。

## 标准流程

```text
扫描相关链接
-> 判断主笔记
-> 合并重复内容
-> 删除或归档孤立笔记
-> 更新资源总览
-> 搜索残留旧链接
-> GitHub 同步
```

## 整理规则

- 一个主题只保留一个主入口。
- 资源索引可以作为章节，不必拆成独立笔记，除非内容足够大。
- 空链接必须尽快补内容或移除。
- 已整理条目从收集箱移到“已整理”。
- 不把 `.obsidian/graph.json`、canvas 草稿等无关文件混进知识提交。
