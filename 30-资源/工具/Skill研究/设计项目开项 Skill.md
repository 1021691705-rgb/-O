# 设计项目开项 Skill

## 一句话总结

把一个模糊设计需求转成可执行项目档案，明确目标、交付物、资料缺口、岗位协同、风险和下一步。

## 参考来源

- Atlassian / Asana 等项目管理资料：开项需要明确目标、范围、角色、交付物和风险。
- 本项目 AI设计公司总控规则：只处理员工管理、设计流程、岗位协同和降本增效。
- Anthropic Skill 思路：把稳定工作流写成可复用步骤。

## 触发场景

- 用户使用 `/开项目`。
- 用户描述一个设计需求但资料不完整。
- 需要启动主图、详情页、海报、AI生图、3D 场景等设计任务。

## 输入

- 项目目标。
- 产品类型。
- 平台 / 渠道。
- 交付尺寸。
- 截止时间。
- 参考图和产品素材。
- 品牌限制和预算。

## 输出

- 项目档案。
- 管理边界。
- 缺失资料清单。
- 初步视觉方向。
- 初始任务看板。
- 首轮风险提醒。

## 标准流程

```text
识别项目类型
-> 明确交付目标
-> 列出已有资料
-> 列出缺失资料
-> 判断涉及岗位
-> 拆第一轮任务
-> 标出风险和成本控制点
-> 给出下一步
```
