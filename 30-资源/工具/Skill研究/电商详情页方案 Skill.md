# 电商详情页方案 Skill

## 一句话总结

把产品卖点、用户痛点和平台规则转成详情页逐屏结构，明确每一屏的销售任务、文案、画面 brief 和素材缺口。

## 参考来源

- Amazon / 电商平台图片与详情页规范：商品图片要准确、清晰，不误导消费者。
- 常见电商详情页逻辑：首屏吸引、痛点共鸣、卖点解释、场景展示、参数证明、信任背书、购买理由。
- AI设计公司总控规则：产品真实性和转化逻辑优先。

## 触发场景

- 用户要做 Amazon、独立站、淘宝、拼多多、小红书电商详情页。
- 用户提供产品但没有详情页结构。
- 需要从主图扩展到完整页面方案。

## 输入

- 产品名称和类型。
- 核心卖点。
- 用户痛点。
- 产品参数、尺寸、材质、认证。
- 品牌调性。
- 竞品链接或截图。
- 平台规则。

## 输出

- 逐屏结构表。
- 每屏销售任务。
- 标题和副标题。
- 画面 brief。
- 所需素材。
- 风险和待确认点。

## 标准流程

```text
确认产品和平台
-> 梳理卖点和痛点
-> 判断购买决策顺序
-> 设计逐屏结构
-> 写标题和副标题
-> 输出画面 brief
-> 标注素材缺口
-> 质检真实性和合规性
```

## 逐屏结构模板

| 屏幕 | 销售任务 | 标题 | 画面 brief | 所需素材 | 风险 |
|---|---|---|---|---|---|
| 首屏 | 建立第一印象 |  | 产品主视觉 | 产品高清图 | 卖点过多 |
| 痛点屏 | 让用户认同问题 |  | 使用前问题场景 | 场景图 | 夸大痛点 |
| 卖点屏 | 解释核心优势 |  | 产品结构 / 功能展示 | 结构图 | 编造参数 |
| 参数屏 | 增加信任 |  | 参数图表 | 官方参数 | 信息不准 |
| 场景屏 | 让用户代入 |  | 真实使用环境 | 场景图 | 产品比例失真 |
