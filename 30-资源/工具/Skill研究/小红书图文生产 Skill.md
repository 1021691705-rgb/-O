# 小红书图文生产 Skill

## 一句话总结

把 Obsidian 笔记、设计经验或项目案例改写成适合小红书发布的图文内容，包括选题、封面、分镜、文案和发布检查。

## 参考来源

- JimLiu `baoyu-skills`：包含小红书图文、信息图、封面、Markdown to HTML、发布类 Skill。
- 小红书内容常见结构：强标题、场景化开头、步骤化干货、图文卡片、互动引导。
- 本公司知识库目标：把内部知识转成可传播内容资产。

## 触发场景

- 用户想把知识库内容发小红书。
- 项目复盘需要转成对外案例。
- Skill 研究、设计流程、降本经验需要做成图文。

## 输入

- Obsidian 主笔记。
- 目标读者。
- 发布目的。
- 可公开范围。
- 图片或案例素材。
- 品牌语气。

## 输出

- 小红书标题。
- 封面文案。
- 6-9 页图文卡片结构。
- 每页正文。
- 配图建议。
- 标签和互动引导。

## 标准流程

```text
确认公开范围
-> 提炼核心观点
-> 写标题和封面
-> 拆图文页
-> 写每页文案
-> 标注配图
-> 检查敏感信息
-> 输出发布版
```

## 图文结构模板

| 页码 | 任务 | 内容 |
|---|---|---|
| 1 | 封面 | 让用户知道为什么要点进来 |
| 2 | 痛点 | 说明常见问题 |
| 3-6 | 方法 | 分步骤讲解 |
| 7 | 案例 | 用真实项目说明 |
| 8 | 总结 | 提炼结论 |
| 9 | 互动 | 引导收藏 / 评论 / 私信 |
